# 指标口径字典

## 数据粒度

- `ecommerce_all`：五个月度表合并后的原始行为日志，一行代表一次行为记录。
- `ecommerce_clean`：全字段去重、时间转换和缺失值标准化后的行为日志。
- `ecommerce_daily_metrics`：日粒度汇总表，一天一行。
- `ecommerce_funnel_summary`：三种漏斗口径汇总表，一种漏斗一行。
- `ecommerce_category_metrics`：品类经营指标表，一个 `category_id` 一行。
- `ecommerce_user_rfm_base`：购买用户 RFM 原始指标表，一个用户一行。
- `ecommerce_user_rfm_scores`：购买用户 RFM 评分表，一个用户一行。
- `ecommerce_user_rfm_segments`：购买用户 RFM 分群明细表，一个用户一行。
- `ecommerce_rfm_segment_summary`：RFM 分群汇总表，一个用户群一行。

## 核心指标

| 指标 | 定义 | 关键口径 |
|---|---|---|
| 总行为数 | 清洗表记录数 | `COUNT(*)` |
| 活跃用户数 | 发生任意行为的去重用户数 | `COUNT(DISTINCT user_id)` |
| 活跃商品数 | 出现过的去重商品数 | `COUNT(DISTINCT product_id)` |
| 总会话数 | 非空去重会话数 | `COUNT(DISTINCT user_session)` |
| 购买用户数 | 至少发生一次 purchase 的去重用户数 | 不限制价格 |
| 购买会话数 | 至少包含一次 purchase 的非空去重会话数 | 不限制价格 |
| 购买事件数 | 清洗后 purchase 行为记录数 | 不是订单量 |
| 有效购买事件数 | purchase 且 `is_valid_price = 1` | 用于金额分母 |
| 交易金额估算 | 有效购买事件 price 之和 | 不是严格 GMV |
| 用户购买转化率 | 购买用户数 / 活跃用户数 | 用户口径 |
| 会话购买转化率 | 购买会话数 / 总会话数 | 会话口径 |
| 平均购买事件金额 | 交易金额估算 / 有效购买事件数 | 不是客单价 |

## 清洗标记

| 字段 | 含义 |
|---|---|
| `is_valid_price = 1` | price > 0，可参与金额计算 |
| `is_valid_price = 0` | price <= 0，保留行为但不参与金额计算 |

## 时间指标

- 日环比：`(当天 - 前一天) / 前一天 × 100%`。
- 7 日移动平均：当前日及前 6 行的简单平均。
- 7 日每日用户转化率均值：每日用户转化率的简单移动平均，不是跨日用户重新去重后的转化率。

## 漏斗指标

| 指标 | 定义 | 关键口径 |
|---|---|---|
| 用户级宽松漏斗 | 同一用户在观察期内出现过 view、cart、purchase | 允许跨会话，不检查顺序 |
| 会话级宽松漏斗 | 同一用户、同一会话内出现过三个阶段 | 不检查顺序 |
| 会话级严格漏斗 | 同一用户、同一会话内首次 view < 首次 cart < 首次 purchase | 使用 `event_time` 比较 |
| 品类宽松转化 | 同一用户在同一 `category_id` 中出现相应行为 | 允许跨会话，不检查顺序 |

## 品类指标

| 指标 | 定义 | 关键口径 |
|---|---|---|
| 品类有效购买事件 | 品类内 purchase 且 `is_valid_price = 1` | 不是订单量 |
| 品类交易金额估算 | 品类内有效购买事件的 price 之和 | 不是严格 GMV |
| 平均有效购买事件金额 | 品类交易金额估算 / 品类有效购买事件数 | 不是客单价 |
| 高流量品类 | `view_users` 位于全部品类前 25% | 使用 `NTILE(4)` |
| 低转化品类 | 品类宽松浏览至购买率低于 10.74% 加权基准 | 用于筛选优先诊断对象 |

## RFM 指标

| 指标 | 定义 | 关键口径 |
|---|---|---|
| R | 观察期结束次日与用户最近购买日期的天数差 | 越小越好 |
| F | 用户去重购买会话数 | 订单频次代理，不是真实订单数 |
| M | 用户有效购买事件 price 之和 | 交易金额估算 |
| R评分 | 按 R 降序计算 `PERCENT_RANK()` 后映射为 1–5 | 最近购买用户得分更高 |
| F评分 | 1次=1分、2次=2分、3次=3分、4–5次=4分、6次及以上=5分 | 避免拆分大量 F=1 用户 |
| M评分 | 按 M 升序计算 `PERCENT_RANK()` 后映射为 1–5 | 高金额用户得分更高 |
